# Day 3: IAM & AWS Security

Identity and Access Management (IAM)
- Framework of policied, processes and technologies that ensure the right individuals and devices have access to the right resources at the exact right times.
- Service to securely control access to AWS resources

IAM Components:
- Users
- Groups (easy for assigning permissions)
- Policies
- Roles

Principle of Least Privilege
- Give only the permissions needed, nothing more.
- Reduces security risk

Why not use Root?
- Because Root is only for administrative task only

Difference between User and Role?
- User are the people inside the IAM
- Roles are identities assumed by service and applications

Why use Multi Factor Authentication?
- Much safer, especially enabled on important accounts like AWS root account
